# Change Log — Report Automation

Maintained alongside every fix/feature made via Claude Code. Each entry records the date, the module affected, the issue reported (or gap found), and what was actually changed. Newest entries at the top. This file lives in the project root, so it's always visible in your folder and downloadable like any other file.

| Date | Module | Issue | Fix |
|------|--------|-------|-----|
| 2026-07-02 | HLA | Bulk/single NGS report types (Single HLA, Transplant Donor, NGS with Photo, 11 Loci, RPL Couple, Single RPL) had no way to set Typing Status to "Incomplete" — it always rendered as "Complete" with no editor control. | Added a "Typing Status" dropdown (Complete / Incomplete) to the Single Entry form and Bulk Editor, alongside the Methodology override field. |
| 2026-07-02 | HLA | IMGT/Coverage section of NGS reports showed a blank, orphaned "Remarks:" label above "Coverage" whenever a patient/donor had any remarks — redundant with the actual remarks text already shown above the locus table. | Removed the static "Remarks:" label from the IMGT/Coverage block; per-person remarks continue to render in their existing location only. |
| 2026-07-01 | HLA | Patient+Result CSV cross-match set `typing_status` to an empty string, rendering "Typing Status:" with no value on the report. | Fixed the parser to default to "Complete" at the source, plus an or-fallback in the report template. |
| 2026-07-01 | HLA | Hospital name "KIMS" was rendered as "Kims" in all reports. | Added "kims" to the known-abbreviation whitelist in `_title_case` so it always renders as "KIMS" even when typed in mixed case or re-cased from an ALL-CAPS patient-form field. |
| 2026-07-01 | HLA | No way to override the Methodology text in single or bulk NGS HLA reports. | Added an optional "Methodology" textarea (labelled "METHODOLOGY (OPTIONAL OVERRIDE)") to the Single Entry form and the Bulk Editor for all NGS report types (Single HLA, Transplant Donor, NGS with Photo, 11 Loci, RPL Couple, Single RPL). Leaving it blank keeps the existing default (MiniSeq text when NABL is on, SurfSeq when off). The value round-trips through draft save/load. |
| 2026-07-01 | HLA | Wide-format Result CSV (BarcodeId/SampleName header) parsed allele columns incorrectly — B-locus values appeared in A-locus, etc. | The CSV's `Comments` column contains thousands of unquoted characters including bare newlines, causing pandas `read_csv` to split each patient row into hundreds of sub-rows and shift all subsequent column values left. Replaced `pd.read_csv` with Python's `csv.reader` (positional column indices); continuation/junk rows that lack a non-empty SampleName and ≥16 columns are skipped automatically. |
| 2026-07-01 | HLA | Importing a SAB Excel file wiped any patient already selected in Single Entry mode (renderManualForm was called unconditionally, resetting all fields). In Bulk mode, it replaced the entire patient list with a single SAB case. | Single Entry: patient data is now captured before the form re-renders; if a patient was already filled in, their details are restored after the form switches to SAB mode and the Excel's patient fields are not overwritten. The form only re-renders when the SAB class (I vs II) actually changes. Bulk: SAB data is matched to an existing patient by PIN or Sample Number and merged in; if no match, the case is appended rather than replacing the list; only when the bulk list is empty does the original create-new-case behaviour apply. |
| 2026-06-30 | PGT-A | Embryo ID containing a dash (e.g. "V-1") was mangled to just "1" in the Results Summary table and embryo detail heading. | The dash-splitting was a render-time heuristic meant only to shorten raw composite bulk-import sample names (e.g. "PATIENT-V1_suffix" → "V1"), but it ran on every Embryo ID regardless of source. Moved the extraction to bulk-import parse time; the renderer now always shows the Embryo ID field verbatim. |
| 2026-06-30 | HLA | "HLA (NGS with Photo)" report's combined patient/donor demography table had no NABL accreditation mark, unlike the Transplant Donor / Single HLA reports. | Added the NABL seal image to the gap column between the patient and donor halves, vertically centered, shown only when the case's NABL flag is true; gap column stays a thin spacer when false. |
| 2026-06-30 | HLA | Dragging a matched patient onto a "Donor" slot in bulk mode copied name/PIN/dates but discarded the patient's actual HLA typing results (one code path explicitly wiped them to empty). | Both donor drag-and-drop handlers now carry over the dragged patient's real `hla` data when present. |
| 2026-06-30 | HLA | Preview crashed with a ReportLab layout error ("too large on page") for some bulk-matched patients. | The wide-format Result CSV's `Comments` column is a raw genotyping-software allele-ambiguity dump (thousands of characters), not a clinical remark. It was being copied verbatim into the patient's Remarks field and overflowing the PDF page. Stopped surfacing that column as remarks. |
| 2026-06-30 | HLA | Patient CSV + Result CSV cross-matching always returned "no patients matched," even with valid overlapping data. | The match function compared `patient.sample_number`, which the CSV patient-list parser never populates (only `pin`/Patient No). Every patient was silently skipped before any real comparison. Fixed to match on `pin`, with a substring fallback for Patient No embedded in a longer sample identifier. |
| 2026-06-30 | HLA | No way to cross-match a separate Patient List CSV against a separate Result CSV (long-format semicolon-quoted or wide-format BarcodeId/SampleName) and show only patients with a result. | Added `parse_result_csv` (auto-detects long vs wide format), `parse_patient_and_result_csv` (cross-match by Patient No), and the `/hla/parse-patient-result-csv` endpoint + a second "Result CSV" upload slot in the bulk tab. |
| 2026-06-30 | PGT-A / NIPT / TERA / Karyotype / HLA | "Select Patient" lookup modal (Find Patients by date range) had no way to filter a long result list. | Added a live-filtering search box (by name or patient no) to the modal across all five report pages, each wired to that page's own JS conventions. |
| 2026-06-30 | PGT-A | Bulk mode: blank/unset Interpretation silently defaulted to "Euploid" instead of "NA," mislabeling abnormal/mosaic embryos with no Conclusion text. | Changed the default fallback to "NA" in three places (bulk per-embryo dropdown, page-1 summary dropdown, payload builder). |
| 2026-06-30 | PGT-A | Patient Information table's grid lines ignored the "Grid lines" dropdown (always forced on after an earlier fix). | Reverted to respecting the toggle — "No Grid" now means no grid anywhere, including Patient Information. |
| 2026-06-30 | PGT-A | "Conditions for reporting mosaicism" section header could get separated from its body paragraph across a page break, leaving an orphaned heading. | Wrapped the header + first paragraph in `KeepTogether` so they always land on the same page. |
| 2026-06-30 | PGT-A | Redundant always-blank "Mosaic calls" row in the CNV status table; the real value already appears in the row above it. | Removed the dead row (and its row-count reservation in the DOCX generator). |
| 2026-06-30 | PGT-A | Custom text typed into Autosomes was silently overwritten with a generic placeholder ("Normal"/"Inconclusive"/etc.) whenever Result classified as Normal/Failed/Inconclusive/Low DNA. | `derive_autosomes()` now preserves custom Autosomes text consistently across all classifications, not just abnormal ones. |
| 2026-06-30 | PGT-A | Result and Interpretation colors were forced from the same shared variable, so e.g. "NA" Interpretation inherited red/blue from an unrelated Result, and Mosaic could render black. | Split into independent color functions (`_classify_color` for Result, `_get_interp_only_color` for Interpretation) in both PDF and DOCX, summary table and detail page. Added explicit color rules per the lab's spec: Interpretation→Autosomes color override (Aneuploid/Chaotic→red, mosaic→blue, Inconclusive→black), "(-)" recognized as red, Mosaic always blue regardless of Autosomes/Sex. |
| 2026-06-30 | PGT-A | "Multiple chromosomal abnormalities" Result Summary was silently rewritten to "Abnormal chromosome complement" on every preview/report. | Backend classification engine mapped that keyword to the wrong internal category (Aneuploid instead of Segmental), which has a different display text. Fixed the mapping. |
| 2026-06-30 | PGT-A | Editing the Interpretation/Result Summary dropdowns and re-generating sometimes still showed "Euploid" / wrong MTcopy despite an explicit different choice. | The "force Euploid when Autosomes+Sex are Normal" safety net (added for bulk-Excel data with inconsistent Conclusion text) was unconditionally overriding explicit user choices. Gated it to only fill in blank/ambiguous interpretations, never override a deliberate selection like Aneuploid/Chaotic/Low level mosaic. |
| 2026-06-30 | PGT-A | DOCX report showed "NA" instead of "Euploid" in the Interpretation column/field for genuinely euploid embryos. | Found and removed an inverted condition in the DOCX generator (summary table + embryo detail page). |
| 2026-06-30 | PGT-A | Result Summary and Interpretation fields in the Results Summary table were read-only; edits in the bulk/manual editor didn't always propagate to preview/report. | Made both fields editable dropdowns (manual + bulk), wired consistently into the same payload used for preview and final generation. |
| 2026-06-30 | PGT-A | Sex Chromosomes field couldn't accept custom values (e.g. "-X", "Mosaic +X") — manual mode had no custom option at all; bulk mode's custom-input box rendered squeezed to nearly invisible. | Replaced with a proper combobox (`<input list>` + `<datalist>`) in both manual and bulk modes — pick from presets or type freely in the same box. Live color-coded to match desktop (black/red/blue). |
| 2026-06-30 | PGT-A (bulk) | Patient/sample matching during Excel import used a single weak substring check with a risky fallback that could silently attach one patient's embryo results to a different patient's report. | Replaced with the desktop app's proven 3-tier match (Sample ID → patient-name prefix → first-name-token) with duplicate-sample tracking; unmatched samples are now dropped instead of misattached. |
| 2026-06-30 | PGT-A | No quick way to duplicate the last embryo's full data into a new entry (existing in the desktop app, missing from web). | Added a "Copy Last Embryo" button to the manual-entry UI. |
