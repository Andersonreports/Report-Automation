NIPS Report Generator — User Guide

Non-Invasive Prenatal Screening

1. Single Entry Tab

Fill in patient demographics and Z-scores. The right pane shows a live PDF preview that auto-refreshes 1 second after you stop typing.

  • Save Draft — save all fields to a JSON file for later restoration
  • Load Draft — restore a previously saved state (also triggers a preview refresh)
  • Generate Report — export PDF and/or DOCX to your output folder
  • Refresh Preview — manually trigger a preview regeneration

2. Batch Upload Tab

Select an .xlsx, .xls, or .csv file. All patients load into the left list. Click any name to open the full inline editor in the centre — you can edit all demographics and Z-scores without leaving this tab. The right pane shows a live preview for the selected patient.

  • Sheet1 — demographics (Patient Name, Sample ID, GA, etc.)
  • Sheet2 (optional) — Z-scores, joined on Sample ID
  • QC column — if present, only rows with QC = "pass" are imported
  • Save All Draft — save all edited batch data as JSON
  • Load Draft — restore a previously saved batch
  • Generate All Reports — export every patient in the batch
  • Save Individual Draft / Generate for This Patient — per-patient actions in the editor

3. Excel Column Mapping

  • Patient Name or Sample Name → patient name
  • Sample ID → sample ID
  • chr1, chromosome1, chrom1 → chr1 Z-score, etc.
  • Fetal DNA, Fetal fraction, FF → fetal fraction (%)

4. Risk Calculation

  • Chromosomes 13, 18, 21: Z-score > 2.8 → High risk
  • Sex chromosomes & other autosomes: Z-score > 6.0 → High risk
  • All others → Low risk

5. Report Comparison Tab

Compare Manual reports (clinician-signed) vs Automated reports (pipeline-generated). In Directory Mode files are matched by filename; in File Mode pick one PDF from each source and a detailed diff is shown.

6. Export Settings

  • PDF / DOCX — choose which formats to export
  • With Branding Logo — toggle Anderson Diagnostics letterhead
  • Output Folder — all generated reports are downloaded from the server to this display path (reports saved server-side)
